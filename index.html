测试

git init 初始化仓库 git status 查看仓库状态

git log 查看提交日志

git add 文件 git管理的是修改,先add再commit

git commit -m “提交说明”

git reset –hard HEAD^ 回滚上个版本 HEAD是个指针,回滚前100个使用HEAD~100.使用git log –pretty=oneline 查看提交的的版本号 git reflog查看命令历史

git reset –hard 版本号 回滚指定版本号

git revert HEAD 撤销当前提交-适合远程推送

git checkout – 文件 可以丢弃工作区的修改

git reset head 文件 可以把暂存区的修改撤销掉,重新放回工作区

git rm 文件 + git commit ：删除版本库文件

git branch 分支名 创建分支

git branch -d 分支名 删除分支

git branch -f 分支名 版本号/HEAD~ 分支切换到指定位置

git checkout 分支名 切换分支

git checkout -b 分支名 创建并切换到分支

git merge name name分支合并到当前分支

git rebase name 复制HEAD指向的分支到name分支下 在共同祖先的基础下,提取HEAD指向的分支的修改:在name分支基础上再应用一次。

git rebase -i HEAD~4 对前4个提交做一个交互式的改动(改变顺序或者删除)

git rebase 分支名A 分支名B 分支B变基到分支A HEAD最终指向分支B
merge与rebase区别:变基是将一系列提交按照原有次序依次应用到另一分支上,而合并是把最终结果合在一起。因此整合的最终结果所指向的快照始终是一样的,只不过提交历史不同罢了。

git cherry-pick 版本号…版本号 复制指定的提交到当前版本号
git tag 标签名 版本名增加指定标签
